# Sports Shop — локальний Django-проєкт

Це невеликий інтернет-магазин спортивних товарів, зроблений для навчального проєкту. Тут реалізовано каталог товарів, кошик, оформлення замовлення, відгуки та простий REST API.

Читайте цей файл, щоб швидко запустити та перевірити проєкт на локальній машині.

## Коротко про зміст
- Каталог товарів з категоріями
- Кошик та оформлення замовлень
- Система відгуків і рейтингів
- Панель адміністратора для управління даними
- REST API (DRF) для основних сутностей

## Швидкий старт (Windows)

1) Створіть та активуйте віртуальне середовище:

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
```

2) Встановіть залежності:

```powershell
pip install -r requirements.txt
```

3) Скопіюйте шаблон змінних оточення і відредагуйте при потребі:

```powershell
copy .env.example .env
# Відкрийте .env і задайте SECRET_KEY, DEBUG тощо
```

4) Застосуйте міграції та створіть суперкористувача:

```powershell
python manage.py migrate
python manage.py createsuperuser
```

5) Запустіть сервер розробки:

```powershell
python manage.py runserver
```

Сайт буде доступний за адресою http://127.0.0.1:8000/, адмін — http://127.0.0.1:8000/admin/.

## Основні можливості
- Реєстрація, вхід/вихід, редагування профілю
- Перегляд товарів за категоріями, пошук і фільтри
- Кошик: додати/змінити/видалити позиції
- Оформлення замовлення з базовою інформацією про доставку
- Система відгуків з оцінками і модерацією
- Проста REST API для інтеграції клієнтів

## Структура проєкту (коротко)

- `sport_shop_project/` — загальні налаштування Django
- `users/` — профілі та аутентифікація
- `products/` — моделі та сторінки каталогу
- `orders/` — кошик і замовлення
- `reviews/` — відгуки користувачів
- `api/` — DRF: серіалізатори та ViewSet'и
- `templates/` — HTML-шаблони
- `static/` — стилі й скрипти

Більше деталей в коді — файли в кожному додатку мають стандартні назви (`models.py`, `views.py`, `admin.py`, `forms.py`, `urls.py`).

## Налаштування середовища
- Значення секретів та `DEBUG` беруться з файлу `.env`.
- Не зберігайте реальний `SECRET_KEY` у публічних репозиторіях.

## API (коротко)
- Базовий маршрут API: `/api/v1/`
- Наявні ресурси: `products`, `categories`, `reviews`, `orders`, `cart`, `users`
- Для захищених операцій потрібна авторизація (через сесію або токен, залежно від конфігурації).

## Що перевірити після запуску
1. Увійти в адмінку і подивитися на категорії/товари.
2. Зареєструвати звичайного користувача і додати товари в кошик.
3. Оформити замовлення — перевірити запис у адмінці.
4. Викликати кілька API-методів через браузер або Postman.
